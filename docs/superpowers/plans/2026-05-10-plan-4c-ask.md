# Plan 4c ‚Äî Ask: RAG over user content ‚Äî Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Ask ‚Äî a natural-language Q&A surface over all user content (items, notes, service records, checklist items, warranties, attachment text) with inline citations. Add an OCR pipeline so phone-photographed receipts and image-only PDFs become indexable. Reuse the existing pgvector extension and the Plan 4b telemetry table.

**Architecture:** One new `Embedding` table (polymorphic across 6 entity types). Two worker jobs ‚Äî `embed-content` for entities and `extract-attachment-text` for OCR. One new Server Action `askQuestion` that embeds the query, runs pgvector cosine top-k, calls Anthropic `messages.parse` with a Zod schema constraining `{ answer, citations[] }`, and logs to `AISuggestionLog` with `kind: 'ASK'`. New `/ask` page renders the answer with clickable citation chips. See `docs/superpowers/specs/2026-05-10-plan-4c-ask-design.md`.

**Tech Stack:** Next.js 16 App Router, TypeScript 6, Prisma 7 + Postgres 16 + pgvector, Auth.js v5, RHF + Zod 4, shadcn (Base UI) components, Voyage AI (`voyage-3.5-lite`, 1024-dim) via raw fetch (no SDK needed), Tesseract.js 7 for OCR, `sharp` for image normalization (already in tree), Anthropic `@anthropic-ai/sdk` (already in tree), pg-boss 12 (existing).

**Spec:** `docs/superpowers/specs/2026-05-10-plan-4c-ask-design.md`

## Conventions for the implementer

These conventions hold across every task. Don't deviate without flagging.

- **Cadence**: each phase below is a separate branch + PR off `main`, in the order listed. Phases C and D may go in parallel (both depend only on Phase B). Don't open later PRs until earlier ones land ‚Äî this plan touches schema, worker boot, and prod env, so stacked branches will pain you.
- **Commits**: signed via 1Password (just `git commit` ‚Äî no `-c user.email=`, no `--no-verify`, no `--no-gpg-sign`). Stage explicit paths, never `git add -A`. Conventional-commits prefixes: `feat(ask):`, `fix(ask):`, `test(ask):`, `docs(ask):`, `chore(ask):`. For attachment OCR work specifically, `feat(ocr):` is also fine.
- **Module-load DATABASE_URL trap**: `lib/db.ts` constructs PrismaClient at module load. Integration tests for new server-only modules must use the dynamic-import-in-`beforeAll` pattern from `tests/integration/notify-job.test.ts`.
- **Env-var trap** (from Plans 3, 4b, 5a): adding any new env var to `lib/env.ts` requires three more edits ‚Äî `.github/workflows/ci.yml` e2e job env block, `Dockerfile` build-step env line (placeholder), and `docker-compose.yml`. This plan adds **three** env vars (`VOYAGE_API_KEY`, `ASK_ENABLED`, `OCR_BACKEND`); Phase B Task 1 covers all three locations upfront.
- **Domain layout**: `lib/ask/{actions.ts,actions.test.ts,retrieve.ts,retrieve.test.ts}`, `lib/embedding/{canonicalize.ts,canonicalize.test.ts,chunk.ts,chunk.test.ts,index.ts,voyage.ts}`, `worker/jobs/{embed-content.ts,extract-attachment-text.ts}`, `lib/ocr/tesseract.ts`. Routes under `app/(app)/ask/`. Components under `components/ask/`. The existing `lib/incoming-email/pdf-text.ts` (PR #101) is the precedent for unpdf text extraction ‚Äî share the helper rather than duplicating it.
- **Dependency policy**: minimal. New deps this plan adds:
  - `tesseract.js@~7.0.0` (Phase D ‚Äî verified currency in conversation)
  - No Voyage SDK; use raw `fetch` against `https://api.voyageai.com/v1/embeddings` (the API surface is one endpoint and a SDK would just wrap it).
  Don't add anything else without re-checking via `pnpm view <pkg>@latest`.
- **shadcn UI primitives available**: same set as the inbound-email plan; no new shadcn components needed.
- **Test database**: integration tests reuse `setupIntegration` + dynamic-import. The pgvector extension is in the existing migration so the test DB already has it.
- **Voyage rate limits**: free tier is 1M tokens/month and 3 req/sec. The backfill job batches up to 128 chunks per request; the per-entity path is one request per chunk-group. Set worker `retryLimit: 5` with exponential backoff; surface persistent 429s via Sentry.

### Task E1 ‚Äî Zod answer schema + prompt

- [ ] **Step 1**: Add to `lib/ai/schemas.ts`:

  export const askCitationSchema = z.object({

- [ ] **Step 1**: Create `lib/ask/actions.ts` exporting `askQuestion(input: { question: string; entityTypes?: ... }): Promise<ActionResult<AskAnswer>>`.
