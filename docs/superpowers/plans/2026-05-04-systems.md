# Systems Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `System` first-class entity that groups items into logical systems (e.g. heat pump + coil + furnace = HVAC), reshape the three event tables (ServiceRecord, Warranty, Reminder) to multi-target via parallel join tables, and add multi-vendor relationships on both items and systems via parallel join tables sharing a role enum. Single PR off `main` on branch `feat/systems`.

**Architecture:** New `System` model sibling to `Item`. Each event table loses its direct `itemId` FK and gains a parallel `*Target` join table with `(parentId, itemId?, systemId?)` rows under a XOR CHECK constraint. Reminder due-state moves down to `ReminderTarget` so completion can advance per-target dates independently. Item and System share a `VendorRole` enum and parallel `ItemVendor`/`SystemVendor` tables that reference either a `Vendor` row or a free-text `freeformName`, again under a XOR CHECK. Vendor FKs on link tables use `onDelete: Restrict` to avoid violating the XOR; vendor deletion is mediated by an explicit user flow.

**Tech Stack:** Next.js 15 App Router, TypeScript 6, Prisma 7 + Postgres 16, Auth.js v5, RHF + Zod, shadcn (Radix-based) components, Vitest 4, Playwright. Worker process (pg-boss) has one query touched.

**Spec:** `docs/superpowers/specs/2026-05-04-systems-design.md`

```ts
import type { TargetInput } from './schema';

interface SystemWithComponents {
  id: string;
  items: Array<{ id: string; archivedAt: Date | null }>;
}

/**
 * When the user checks a system in the picker, also yield all of its
 * active component items. Items already in `seed` are kept; the system
 * itself is included. Returns a deduplicated, ordered TargetInput[].
 */
export function expandSystemSelection(
  seed: TargetInput[],
  system: SystemWithComponents,
): TargetInput[] {
  const seen = new Set(seed.map((t) => t.itemId ?? `sys:${t.systemId}`));
  const out = [...seed];
  if (!seen.has(`sys:${system.id}`)) {
    out.push({ systemId: system.id });
    seen.add(`sys:${system.id}`);
  }
  for (const item of system.items) {
    if (item.archivedAt) continue;
    if (seen.has(item.id)) continue;
    out.push({ itemId: item.id });
    seen.add(item.id);
  }
  return out;
}
```

```ts
import { Decimal } from 'decimal.js';

interface RollupInput {
  installCost: Decimal | null;
  components: Array<{ purchasePrice: Decimal | null; archivedAt: Date | null }>;
}

export function computeCostRollup(input: RollupInput): {
  componentsSubtotal: Decimal;
  installCost: Decimal;
  total: Decimal;
  hasAnyData: boolean;
} {
  const subtotal = input.components.reduce(
    (acc, c) => (c.archivedAt || !c.purchasePrice ? acc : acc.plus(c.purchasePrice)),
    new Decimal(0),
  );
  const install = input.installCost ?? new Decimal(0);
  const total = subtotal.plus(install);
  const hasAnyData = !subtotal.isZero() || !install.isZero();
  return { componentsSubtotal: subtotal, installCost: install, total, hasAnyData };
}
```
