# Phase 2: axe-core Page-Level a11y Scans — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hard-gate WCAG 2.1 AA accessibility across the whole app by scanning every route (empty + populated, desktop + mobile) with axe-core in the existing Playwright harness.

**Architecture:** Extract the route lists + data-seeding from `screenshots.spec.ts` into a shared `tests/e2e/_routes.ts` (so the screenshot helper and the new a11y spec never drift). A new `tests/e2e/a11y.spec.ts` signs in once, walks the routes at both viewports, and runs `AxeBuilder` with the WCAG 2.1 AA tag set, asserting zero violations (minus a documented exclusions allowlist). The work is **measurement-driven**: scaffold → measure the backlog → triage/fix → wire the CI gate.

**Tech Stack:** `@axe-core/playwright` 4.11.3, Playwright 1.60 (chromium, serialized workers), the mock-OIDC e2e harness, GitHub Actions.

**Spec:** `docs/superpowers/specs/2026-05-22-a11y-axe-scans-design.md`

export async function seedPopulated(
  page: Page,
  opts?: { onSuggestInterstitial?: (page: Page) => Promise<void> },
): Promise<SeededUrls> {
  // Vendor
  await page.goto('/vendors/new');
  await page.getByLabel('Name').fill('Acme HVAC Services');
  await page.getByRole('button', { name: /^(Create|Save)/ }).first().click();
  await page.waitForURL(/\/vendors\/c[a-z0-9]+$/);

  // System
  await page.goto('/systems/new');
  await page.getByLabel('Name').fill('Heating');
  await page.getByRole('button', { name: /^(Create|Save)/ }).first().click();
  await page.waitForURL(/\/systems\/c[a-z0-9]+$/);

  // Item (pause on the suggest-after-create interstitial for the optional hook)
  await page.goto('/items/new');
  await page.getByLabel('Name').fill('Furnace');
  await page.getByRole('combobox', { name: 'Category' }).click();
  await page.getByRole('option', { name: /HVAC/i }).click();
  await page.getByRole('button', { name: 'Create item' }).click();
  await expect(page).toHaveURL(/\/items\/c[a-z0-9]+\/suggest-after-create$/);
  if (opts?.onSuggestInterstitial) await opts.onSuggestInterstitial(page);
  await page.getByRole('button', { name: 'Skip' }).click();
  await page.waitForURL(/\/items\/c[a-z0-9]+$/);
  const itemUrl = page.url();
  const itemId = itemUrl.match(/\/items\/(c[a-z0-9]+)/)?.[1];

  // Service record
  await page.goto(`/service/new?itemId=${itemId}`);
  await page.getByLabel('Performed on').fill('2026-04-15');
  await page.getByLabel('Summary').fill('Annual tune-up');

  await page.goto('/reminders/new');
  await page.getByLabel('Title').fill('Change furnace filter');
  await page.getByLabel(/Due/).first().fill('2026-06-01');
  await page.getByRole('button', { name: /^(Create|Save)/ }).first().click();
  await page.waitForURL(/\/reminders\/c[a-z0-9]+$/);

  const noteUrl = page.url();

  return { itemUrl, serviceUrl, reminderUrl, noteUrl };
}

- [ ] **Step 2: Create `tests/e2e/a11y.spec.ts`**

```ts
import AxeBuilder from '@axe-core/playwright';
import { expect, type Page, test } from '@playwright/test';
import { resetAuth, signIn } from './auth';
import { A11Y_EXCLUDED_RULES } from './a11y-exclusions';
import { EMPTY_ROUTES, populatedRoutes, type SeededUrls, seedPopulated, VIEWPORTS } from './_routes';

const WCAG_AA = ['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa'];

async function scan(page: Page, label: string) {
  // Guard against silent auth bounce / redirect producing a false "0 violations".
  await page.waitForLoadState('domcontentloaded');
  expect(page.url(), `${label}: redirected to sign-in`).not.toMatch(/\/api\/auth\/signin/);

  let builder = new AxeBuilder({ page }).withTags(WCAG_AA);
  if (A11Y_EXCLUDED_RULES.length > 0) builder = builder.disableRules(A11Y_EXCLUDED_RULES);
  const { violations } = await builder.analyze();

  const summary = violations
    .map((v) => `  [${v.impact}] ${v.id}: ${v.help} (${v.nodes.length} node(s))\n    ${v.helpUrl}\n    ${v.nodes.map((n) => n.target.join(' ')).join('\n    ')}`)
    .join('\n');
  expect(violations, `${label} a11y violations:\n${summary}`).toEqual([]);
}

test.describe('accessibility (WCAG 2.1 AA)', () => {
  test('empty-state routes', async ({ page, context }) => {
    test.setTimeout(300_000);
    await resetAuth();
    await context.clearCookies();
    await signIn(page);
    for (const vp of VIEWPORTS) {
      await page.setViewportSize({ width: vp.width, height: vp.height });
      for (const route of EMPTY_ROUTES) {
        await page.goto(route.path);
        await scan(page, `${route.name} [${vp.name}]`);
      }
    }
  });

  test('populated routes', async ({ page, context }) => {
    test.setTimeout(300_000);
    await resetAuth();
    await context.clearCookies();
    await signIn(page);
    const urls: SeededUrls = await seedPopulated(page);
    for (const vp of VIEWPORTS) {
      await page.setViewportSize({ width: vp.width, height: vp.height });
      for (const route of populatedRoutes(urls)) {
        await page.goto(route.path);
        await scan(page, `${route.name} [${vp.name}]`);
      }
    }
  });
});
```
Add to `package.json` scripts: `"test:a11y": "playwright test tests/e2e/a11y.spec.ts"` (mirrors `test:e2e` — needs `AUTH_OIDC_ISSUER` env, supplied by the CI job) and `"test:a11y:local": "bash tests/e2e/run-local.sh tests/e2e/a11y.spec.ts"` (local run against the dev DB, per spec parity). Each axe scan is a single `expect` assertion (fast), and the per-test `test.setTimeout(300_000)` covers the 74-scan loop; no global-timeout concern.

## Notes & Risks

- **Measurement-driven:** Task 3's size is unknown until Task 2 runs. The Task 2 decision gate prevents an open-ended PR — surface a large backlog rather than grind or silently suppress.
- **Documented exclusions only:** never blanket-disable a rule to go green; every suppression carries a written reason. This keeps the hard gate meaningful.
- **Runtime:** ~24 empty + 13 populated routes × 2 viewports ≈ 74 scans + seeding — a few minutes in the dedicated `a11y` CI job; the `e2e` job stays `@critical`-only.
- **Per-task commits are safe** — each task compiles/tests independently (Task 2 commits a not-yet-gated spec; the gate lands in Task 4 only after Task 3 makes it green).
- **`AxeBuilder` default import:** `import AxeBuilder from '@axe-core/playwright'` (default export) for 4.x.
